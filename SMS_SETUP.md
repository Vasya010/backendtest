# Настройка SMS для отправки кодов подтверждения

## Вариант 1: Twilio (рекомендуется, работает везде)

1. Зарегистрируйтесь на https://www.twilio.com/
2. Получите:
   - Account SID
   - Auth Token
   - Номер телефона для отправки SMS
3. Добавьте в файл `.env` в папке `backend`:

```
SMS_SERVICE=twilio
TWILIO_ACCOUNT_SID=ваш_account_sid
TWILIO_AUTH_TOKEN=ваш_auth_token
TWILIO_PHONE_NUMBER=+1234567890
```

4. Перезапустите сервер

**Преимущества Twilio:**
- Работает по всему миру
- Надежный сервис
- Хорошая документация
- Есть бесплатный пробный период

## Вариант 2: SMS.ru (для стран СНГ)

1. Зарегистрируйтесь на https://sms.ru/
2. Получите API ID в личном кабинете
3. Добавьте в файл `.env`:

```
SMS_SERVICE=smsru
SMS_API_ID=ваш_api_id_от_sms_ru
```

4. Перезапустите сервер

## Вариант 3: Telegram (для тестирования)

1. Узнайте свой chat_id (напишите боту @userinfobot)
2. Добавьте в файл `.env`:

```
SMS_SERVICE=telegram
ADMIN_TELEGRAM_CHAT_ID=ваш_chat_id
TELEGRAM_BOT_TOKEN=ваш_токен_бота
```

## Формат номера телефона

Номер должен быть в международном формате:
- Правильно: `+996555123456` или `996555123456`
- Система автоматически очистит и отформатирует номер

## Автоматический выбор сервиса

Система автоматически попробует использовать:
1. Выбранный сервис (SMS_SERVICE)
2. Если не работает - попробует Twilio
3. Если не работает - попробует SMS.ru
4. В крайнем случае - отправит в Telegram

## Тестирование

Для тестирования без реальной отправки SMS:
- Код будет выводиться в консоль сервера
- Код будет показываться в приложении (для разработки)

## Важно

В продакшене уберите возврат кода в ответе API (строка `code: code` в `send-code` endpoint).
